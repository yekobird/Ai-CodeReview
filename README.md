# 🔍 AI Code Review

基于 Python 的 Git 提交自动代码审查工具，提交代码后自动触发审查并生成报告。

## 🎯 功能

- **自动触发**：`git commit` 后自动执行代码审查
- **多维度规则**：6 大类 25+ 条审查规则
- **分级报告**：阻断 / 严重 / 警告 / 建议 四级分类
- **本地运行**：无需网络，零依赖 AI 模型
- **可控开关**：支持随时开启/关闭审查

## 📊 审查规则

| 类别 | 数量 | 级别 | 示例 |
|------|:--:|------|------|
| 安全 | 4 条 | 阻断/严重 | SQL注入、硬编码密码 |
| 质量 | 5 条 | 严重/警告 | 空指针、异常吞没、资源泄漏 |
| 性能 | 3 条 | 警告 | 循环内查库、字符串拼接 |
| 规范 | 4 条 | 建议 | 命名、注释、方法长度 |
| 设计 | 2 条 | 建议 | 单一职责、废弃API |
| 兼容性 | 1 条 | 严重 | 硬编码路径 |

## 📁 文件结构
├── review/
│   ├── __init__.py          # 包初始化（空文件）
│   ├── config.json          # 审查开关配置
│   ├── rules.py             # 审查规则库（6大类规则定义）
│   └── agent.py             # 主程序（diff获取、规则匹配、报告生成）
├── reports/                 # 审查报告输出目录
├── .git/hooks/post-commit   # Git钩子（提交后自动触发）
└── README.md



## 📝 文件说明

**review/rules.py** - 审查规则库，定义了所有审查规则。每条规则包含规则ID、类别、严重级别、正则匹配模式、问题描述和修复建议。

**review/agent.py** - 主审查程序，负责获取最近一次 Git 提交的代码变更、提取新增代码行、执行规则匹配、生成 Markdown 格式审查报告、保存报告到 reports/ 目录。

**review/config.json** - 审查开关配置，true 开启，false 关闭。

**.git/hooks/post-commit** - Git 提交后钩子，自动调用 python review/agent.py。
